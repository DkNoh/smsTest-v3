# SMS V3 Codex 작업 규칙

이 저장소는 SMS/LMS/알림톡 대량 발송 및 이력 관리 시스템의 v3 BASE PROJECT를 만들기 위한 작업 공간이다.

현재 단계의 목표는 화면을 많이 만드는 것이 아니라, 폐쇄망에서도 반복 생성과 검증이 가능한 BASE 구조를 먼저 완성하는 것이다.

## 필수 읽기 순서

1. `docs/base/README.md`
2. `docs/base/rules-index.md`

이후 작업별 상세 읽기 순서는 `docs/base/rules-index.md`를 단일 기준으로 따른다.

## 핵심 원칙

- 사용자 기준 테이블은 실제 운영 구조인 `EMP`, 부서 기준 테이블은 `DEP`를 사용한다.
- `EMP`의 기본키는 `(EMP_ID, DEP_ID)` 복합키다. 사용자 식별, 권한, 감사로그, 메뉴 권한은 `EMP_ID` 단독으로 설계하지 않는다.
- `EMP.PERM_*` 컬럼은 v3 권한 판단에 사용하지 않는다. legacy 참고값으로만 취급한다.
- local/dev/prod의 차이는 인증 방식뿐이다. 권한 판단은 모든 환경에서 동일한 v3 권한 테이블을 사용한다.
- local은 LDAP을 사용하지 않고 ID만 입력해 로그인한다. 단, `EMP.ACT_YN = 'Y'`인 사용자 1건이 확인되어야 한다.
- dev/prod는 LDAP 인증을 사용한다. 현재 LDAP 정보는 placeholder로만 둔다.
- 메뉴는 v2 운영 메뉴와 동일한 구조를 v3 메뉴 테이블 seed 기준으로 삼는다.
- 메뉴 source는 `sms.menu.source=static|db`처럼 명시적으로 선택한다. DB 실패 시 static으로 자동 대체하지 않는다.
- 개발자가 폐쇄망 DB에 직접 테이블을 만들 수 없으므로 DBA 전달용 DDL 문서를 반드시 유지한다.

## 구현 품질

- 운영 중 바뀔 값(타임아웃, 경로, 한도)은 `application*.yml`로 외부화한다.
- 매직넘버/매직스트링은 상수로 둔다.
- 같은 로직이 2곳 이상이면 메서드로 추출한다.
- 함수 하나는 한 가지 일만 한다.
- 파일이 300줄을 초과하면 분리를 검토한다.

## 완료 기준

코드 작업이 추가되는 단계부터는 다음 조건을 만족해야 완료로 보고한다.

- `mvn test` 성공
- `mvn -DskipTests package` 성공
- 관련 문서 갱신
- local/dev/prod profile별 설정 검토

테스트나 빌드를 실행하지 못한 경우 완료가 아니라 부분 완료로 보고한다.

## 금지

- 실패를 fallback으로 우회하지 않는다.
- 인증/권한 오류를 try-catch로 삼키지 않는다.
- 실제 DB 구조 확인 없이 컬럼명을 확정하지 않는다.
- 지시받지 않은 패턴/라이브러리/의존성을 임의로 도입하지 않는다. 필요하면 먼저 승인을 받는다.
- 운영(prod) 서버 또는 운영 DB 작업을 사전 승인 없이 진행하지 않는다.
- 운영 비밀번호, LDAP 비밀번호, DB 접속 비밀번호를 문서나 코드에 기록하지 않는다.
- `EMP.PERM_*`를 신규 권한 로직의 기준으로 사용하지 않는다.
