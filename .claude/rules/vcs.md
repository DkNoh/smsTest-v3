# 버전관리 / 커밋 규칙

- 커밋 전 `git status --short`로 변경 파일을 확인한다.
- 관련 없는 변경을 한 커밋에 섞지 않는다.
- `.env.local`, `target/`, 비밀값 파일이 staged에 없는지 커밋 전에 확인한다.
- 메시지 형식은 `feat|fix|refactor|docs|chore(scope): 한 줄 요약`을 사용한다.
- 커밋 전 변경 파일 목록과 커밋 메시지 초안을 사용자에게 공유하고 승인 후 커밋한다.
- force push는 사용자가 명시적으로 요청한 경우에만 한다.
- 화면 1개 생성 = 커밋 1개를 기본 단위로 한다 (scaffold 산출물 + 테스트 + 메뉴 SQL).
- BASE 공통 코드 수정과 화면 생성을 한 커밋에 섞지 않는다.
- BASE 공통 코드를 수정한 커밋에는 `mvn test` PASS를 전제로 한다.
