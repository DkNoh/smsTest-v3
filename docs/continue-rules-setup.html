Continue 룰 적용 가이드

.claude/rules → .continue/rules 로 기존 작업 룰을 그대로 이관하기

폐쇄망 STS에서 Continue 확장이 v3의 기존 작업 룰을 그대로 쓰도록 맞추는 문서입니다. 인터넷 불필요 — 전부 프로젝트 내 로컬 파일입니다. 핵심은 frontmatter의 paths: 를 globs: 로 바꾸는 것이 거의 전부라는 점입니다.

.claude/rules/*.md
paths: → .continue/rules/*.md
globs: + alwaysApply

1. 폴더 구조

.claude/rules/ 와 1:1로 대응하는 .continue/rules/ 를 만듭니다.

<프로젝트루트>/
├─ .claude/rules/          (기존, Claude Code용 — 그대로 둔다)
└─ .continue/
   └─ rules/               (신규, Continue용)
      ├─ 00-core.md            ← CLAUDE.md 핵심을 항상 적용
      ├─ project.md            ← 항상 적용
      ├─ vcs.md                ← 항상 적용
      ├─ environment-auth.md   ← globs 한정
      ├─ menu-authority.md     ← globs 한정
      ├─ mybatis-oracle.md     ← globs 한정
      ├─ scaffold-query.md     ← globs 한정
      ├─ testing.md            ← globs 한정
      └─ thymeleaf.md          ← globs 한정
왜 두 폴더를 둘 다 두나? .claude/rules는 Claude Code가, .continue/rules는 Continue가 읽습니다. 폴더를 공유하지 않으므로 복사해서 형식만 맞춥니다.

2. 변환 규칙 (frontmatter)

항목.claude/rules (현재).continue/rules (변환 후)
적용 범위paths: (glob 리스트)globs: (glob 리스트) — 이름만 변경
범위 한정 룰paths: 있음globs: + alwaysApply: false
공통 룰frontmatter 없음alwaysApply: true
본문마크다운그대로 복사 (수정 불필요)

Continue frontmatter가 인식하는 키: name, description, globs, alwaysApply.

  • alwaysApply: true → 매 요청에 항상 포함 (공통 룰)
  • alwaysApply: false + globs → context에 해당 파일이 있을 때만 포함 (범위 룰)

4. 예시 — 범위 한정 룰 (mybatis-oracle.md)

paths: 를 globs: 로 바꾸고 alwaysApply: false 만 추가. 본문은 그대로.

---
name: MyBatis / Oracle Rules
description: MyBatis XML과 Oracle SQL 작성 규칙
globs:
  - "**/mapper/**/*.xml"
  - "**/resources/mapper/**/*.xml"
  - "**/db/oracle/**/*.sql"
alwaysApply: false
---

# MyBatis / Oracle Rules

- Oracle 19c 기준으로 작성한다.
- 실제 DB Object 확인 없이 컬럼명을 확정하지 않는다.
- `SELECT *`를 사용하지 않는다.
(... 기존 본문 그대로 ...)

5. 예시 — 공통 룰 (vcs.md)

frontmatter가 없던 파일은 alwaysApply: true 만 붙입니다.

---
name: VCS / Commit Rules
description: 버전관리 / 커밋 공통 규칙
alwaysApply: true
---

# 버전관리 / 커밋 규칙

- 커밋 전 `git status --short`로 변경 파일을 확인한다.
(... 기존 본문 그대로 ...)

6. 신규 — 00-core.md (CLAUDE.md 핵심 이관)

CLAUDE.md의 핵심 원칙·완료 판정·DB 안전은 Continue에 자동 로드되지 않으므로 항상 적용 룰로 옮깁니다.

---
name: Core Principles
description: 프로젝트 최상위 작업 원칙
alwaysApply: true
---

# 핵심 원칙

1. fallback 금지: 실패하면 원인을 수정한다.
2. 우회/꼼수 금지, 에러 숨김 금지: 에러는 즉시 전파한다.
3. 지시하지 않은 패턴/라이브러리 임의 도입 금지.
4. 같은 에러 3회 반복 시 중단하고 원문·시도·추정원인을 보고한다.

# 완료 판정
- "완료"는 `mvn test` PASS + `mvn -DskipTests package` 성공일 때만 사용한다.
- 미검증 상태는 "부분 완료"로 보고한다.

# DB / 서버 안전
- 운영 DB 작업은 사전 승인. 작업 전 active profile을 확인한다.
- 비밀값은 코드/공용 문서에 두지 않는다 (.env.local, CLAUDE.local.md).

7. 주의사항

인코딩 — 룰 본문이 한글이므로 파일을 반드시 UTF-8로 저장합니다. STS에서 새 파일 생성 시 인코딩을 확인하세요.
  • .gitignore .continue/ 를 형상관리할지 결정합니다. 팀 공유면 커밋, 개인 설정이면 ignore.
  • globs 기본 동작 globs만 있고 alwaysApply를 생략하면 "매칭 파일에만 적용"이 기본입니다. 명시적으로 alwaysApply: false를 적어 의도를 분명히 합니다.
  • testing.md globs가 **/*.java 등 사실상 전 파일이라 거의 항상 걸립니다. 부담되면 alwaysApply: true로 바꿔도 결과는 비슷합니다.
  • config.yaml Continue는 .continue/config.yaml에서 룰을 참조할 수도 있으나, 위처럼 rules/ 폴더에 파일로 두는 방식이 .claude/rules와 1:1이라 유지보수가 쉽습니다.
요약 — 핵심은 paths: → globs: 한 줄 변환 + 공통 룰(project / vcs / core)에 alwaysApply: true 붙이기. 본문은 그대로 복붙하면 됩니다.
