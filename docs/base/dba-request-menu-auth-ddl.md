# DBA 요청용 메뉴/권한 테이블 DDL

이 문서는 v3 BASE PROJECT에서 필요한 신규 메뉴/역할/권한 테이블 생성 요청서다.

## 전제

- Schema: `SMS`
- Oracle: 19c
- 기존 사용자/부서 테이블: `EMP`, `DEP`
- 신규 생성 대상: `TB_ROLE`, `TB_EMP_ROLE`, `TB_MENU`, `TB_MENU_AUTH`
- 이 문서에는 `DROP TABLE`을 포함하지 않는다.
- 실제 운영 반영 전 DBA 검토가 필요하다.

## 권한 모델 요약

```text
EMP / DEP       : 기존 운영 사용자/부서 기준 테이블
TB_ROLE         : v3 역할 코드
TB_EMP_ROLE     : EMP 복합키 기준 직원-역할 매핑
TB_MENU         : v2 운영 메뉴와 동일한 메뉴 트리
TB_MENU_AUTH    : 역할별 메뉴 기능 권한
```

`EMP.PERM_*` 컬럼은 v3 런타임 권한 판단에 사용하지 않는다.

## 1. TB_ROLE

```sql
CREATE TABLE SMS.TB_ROLE (
    ROLE_CD    VARCHAR2(30)  NOT NULL,
    ROLE_NM    VARCHAR2(100) NOT NULL,
    ROLE_DESC  VARCHAR2(500),
    SORT_ORD   NUMBER(5)     DEFAULT 0 NOT NULL,
    USE_YN     CHAR(1)       DEFAULT 'Y' NOT NULL,
    REG_ID     VARCHAR2(12)  NOT NULL,
    REG_DTTM   TIMESTAMP(6)  DEFAULT SYSTIMESTAMP NOT NULL,
    UPD_ID     VARCHAR2(12),
    UPD_DTTM   TIMESTAMP(6),
    CONSTRAINT PK_TB_ROLE PRIMARY KEY (ROLE_CD),
    CONSTRAINT CK_TB_ROLE_USE_YN CHECK (USE_YN IN ('Y', 'N'))
);

COMMENT ON TABLE SMS.TB_ROLE IS 'v3 역할 코드';
COMMENT ON COLUMN SMS.TB_ROLE.ROLE_CD IS '역할 코드';
COMMENT ON COLUMN SMS.TB_ROLE.ROLE_NM IS '역할명';
COMMENT ON COLUMN SMS.TB_ROLE.USE_YN IS '사용 여부';
```

## 2. TB_EMP_ROLE

```sql
CREATE TABLE SMS.TB_EMP_ROLE (
    EMP_ID    VARCHAR2(12) NOT NULL,
    DEP_ID    VARCHAR2(12) NOT NULL,
    ROLE_CD   VARCHAR2(30) NOT NULL,
    USE_YN    CHAR(1)      DEFAULT 'Y' NOT NULL,
    REG_ID    VARCHAR2(12) NOT NULL,
    REG_DTTM  TIMESTAMP(6) DEFAULT SYSTIMESTAMP NOT NULL,
    UPD_ID    VARCHAR2(12),
    UPD_DTTM  TIMESTAMP(6),
    CONSTRAINT PK_TB_EMP_ROLE PRIMARY KEY (EMP_ID, DEP_ID, ROLE_CD),
    CONSTRAINT FK_TB_EMP_ROLE_EMP FOREIGN KEY (EMP_ID, DEP_ID)
        REFERENCES SMS.EMP (EMP_ID, DEP_ID),
    CONSTRAINT FK_TB_EMP_ROLE_ROLE FOREIGN KEY (ROLE_CD)
        REFERENCES SMS.TB_ROLE (ROLE_CD),
    CONSTRAINT CK_TB_EMP_ROLE_USE_YN CHECK (USE_YN IN ('Y', 'N'))
);

CREATE INDEX SMS.IX_TB_EMP_ROLE_01 ON SMS.TB_EMP_ROLE (ROLE_CD);
CREATE INDEX SMS.IX_TB_EMP_ROLE_02 ON SMS.TB_EMP_ROLE (DEP_ID, EMP_ID);

COMMENT ON TABLE SMS.TB_EMP_ROLE IS 'v3 직원-역할 매핑';
COMMENT ON COLUMN SMS.TB_EMP_ROLE.EMP_ID IS '직원 ID. EMP.EMP_ID';
COMMENT ON COLUMN SMS.TB_EMP_ROLE.DEP_ID IS '부서 ID. EMP.DEP_ID';
COMMENT ON COLUMN SMS.TB_EMP_ROLE.ROLE_CD IS '역할 코드. TB_ROLE.ROLE_CD';
```

## 3. TB_MENU

```sql
CREATE TABLE SMS.TB_MENU (
    MENU_ID         VARCHAR2(50)  NOT NULL,
    PARENT_MENU_ID  VARCHAR2(50),
    MENU_NM         VARCHAR2(100) NOT NULL,
    MENU_URL        VARCHAR2(300),
    MENU_LEVEL      NUMBER(2)     NOT NULL,
    SORT_ORD        NUMBER(5)     DEFAULT 0 NOT NULL,
    MENU_TYPE       CHAR(1)       DEFAULT 'M' NOT NULL,
    ICON_NM         VARCHAR2(50),
    DISPLAY_YN      CHAR(1)       DEFAULT 'Y' NOT NULL,
    USE_YN          CHAR(1)       DEFAULT 'Y' NOT NULL,
    SYSTEM_YN       CHAR(1)       DEFAULT 'N' NOT NULL,
    REMARK          VARCHAR2(500),
    REG_ID          VARCHAR2(12)  NOT NULL,
    REG_DTTM        TIMESTAMP(6)  DEFAULT SYSTIMESTAMP NOT NULL,
    UPD_ID          VARCHAR2(12),
    UPD_DTTM        TIMESTAMP(6),
    CONSTRAINT PK_TB_MENU PRIMARY KEY (MENU_ID),
    CONSTRAINT FK_TB_MENU_PARENT FOREIGN KEY (PARENT_MENU_ID)
        REFERENCES SMS.TB_MENU (MENU_ID),
    CONSTRAINT CK_TB_MENU_TYPE CHECK (MENU_TYPE IN ('G', 'M')),
    CONSTRAINT CK_TB_MENU_DISPLAY_YN CHECK (DISPLAY_YN IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_USE_YN CHECK (USE_YN IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_SYSTEM_YN CHECK (SYSTEM_YN IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_URL_BY_TYPE CHECK (
        (MENU_TYPE = 'G' AND MENU_URL IS NULL)
        OR (MENU_TYPE = 'M' AND MENU_URL IS NOT NULL)
    )
);

CREATE UNIQUE INDEX SMS.UX_TB_MENU_URL ON SMS.TB_MENU (MENU_URL);
CREATE INDEX SMS.IX_TB_MENU_01 ON SMS.TB_MENU (PARENT_MENU_ID, SORT_ORD);
CREATE INDEX SMS.IX_TB_MENU_02 ON SMS.TB_MENU (USE_YN, DISPLAY_YN);

COMMENT ON TABLE SMS.TB_MENU IS 'v3 메뉴 트리';
COMMENT ON COLUMN SMS.TB_MENU.MENU_ID IS '메뉴 ID';
COMMENT ON COLUMN SMS.TB_MENU.PARENT_MENU_ID IS '상위 메뉴 ID';
COMMENT ON COLUMN SMS.TB_MENU.MENU_NM IS '메뉴명';
COMMENT ON COLUMN SMS.TB_MENU.MENU_URL IS '화면 URL';
COMMENT ON COLUMN SMS.TB_MENU.MENU_TYPE IS 'G: 그룹, M: 화면 메뉴';
```

## 4. TB_MENU_AUTH

```sql
CREATE TABLE SMS.TB_MENU_AUTH (
    MENU_ID        VARCHAR2(50) NOT NULL,
    ROLE_CD        VARCHAR2(30) NOT NULL,
    CAN_READ       CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_CREATE     CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_UPDATE     CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_DELETE     CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_APPROVE    CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_CANCEL     CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_DOWNLOAD   CHAR(1)      DEFAULT 'N' NOT NULL,
    CAN_MASK_VIEW  CHAR(1)      DEFAULT 'N' NOT NULL,
    USE_YN         CHAR(1)      DEFAULT 'Y' NOT NULL,
    REG_ID         VARCHAR2(12) NOT NULL,
    REG_DTTM       TIMESTAMP(6) DEFAULT SYSTIMESTAMP NOT NULL,
    UPD_ID         VARCHAR2(12),
    UPD_DTTM       TIMESTAMP(6),

    CONSTRAINT FK_TB_MENU_AUTH_MENU FOREIGN KEY (MENU_ID)
        REFERENCES SMS.TB_MENU (MENU_ID),
    CONSTRAINT FK_TB_MENU_AUTH_ROLE FOREIGN KEY (ROLE_CD)
        REFERENCES SMS.TB_ROLE (ROLE_CD),
    CONSTRAINT CK_TB_MENU_AUTH_READ CHECK (CAN_READ IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_CREATE CHECK (CAN_CREATE IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_UPDATE CHECK (CAN_UPDATE IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_DELETE CHECK (CAN_DELETE IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_APPROVE CHECK (CAN_APPROVE IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_CANCEL CHECK (CAN_CANCEL IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_DOWNLOAD CHECK (CAN_DOWNLOAD IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_MASK CHECK (CAN_MASK_VIEW IN ('Y', 'N')),
    CONSTRAINT CK_TB_MENU_AUTH_USE_YN CHECK (USE_YN IN ('Y', 'N'))
);

CREATE INDEX SMS.IX_TB_MENU_AUTH_01 ON SMS.TB_MENU_AUTH (ROLE_CD, USE_YN);

COMMENT ON TABLE SMS.TB_MENU_AUTH IS 'v3 역할별 메뉴 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_READ IS '화면 접근 및 조회 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_CREATE IS '신규 등록 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_UPDATE IS '수정 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_DELETE IS '삭제 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_APPROVE IS '승인/반려 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_CANCEL IS '취소 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_DOWNLOAD IS '다운로드 권한';
COMMENT ON COLUMN SMS.TB_MENU_AUTH.CAN_MASK_VIEW IS '개인정보 비마스킹 조회 권한';
```

## 6. 메뉴 seed

메뉴 seed는 `docs/base/v2-menu-baseline.md`의 메뉴 목록을 기준으로 작성한 `db/oracle/02_menu_auth_seed.sql`을 사용한다.

운영 반영 시에는 다음 순서로 실행한다.

1. `TB_ROLE`
2. `TB_MENU`
3. `TB_EMP_ROLE`
4. `TB_MENU_AUTH`

`TB_MENU_AUTH` 초기값은 최소한 `ROLE_ADMIN`에 모든 메뉴 권한을 부여한다.

```sql
INSERT INTO SMS.TB_MENU_AUTH (
    MENU_ID, ROLE_CD,
    CAN_READ, CAN_CREATE, CAN_UPDATE, CAN_DELETE,
    CAN_APPROVE, CAN_CANCEL, CAN_DOWNLOAD, CAN_MASK_VIEW,
    USE_YN, REG_ID
)
SELECT
    MENU_ID,
    'ROLE_ADMIN',
    'Y', 'Y', 'Y', 'Y',
    'Y', 'Y', 'Y', 'Y',
    'Y', 'SYSTEM'
FROM SMS.TB_MENU
WHERE USE_YN = 'Y';
```

## 7. DBA 확인 요청사항

- `EMP(EMP_ID, DEP_ID)` 기본키가 실제 운영 DB에 존재하는지 확인한다.
- `TB_EMP_ROLE`에서 `SMS.EMP`로 외래키 생성이 가능한지 확인한다.
- 운영 표준상 `TIMESTAMP(6)` 사용이 가능한지 확인한다. 불가능하면 `CHAR(14)` 또는 `VARCHAR2(14)` 정책으로 조정한다.
- 테이블스페이스 지정이 필요한 경우 운영 표준 테이블스페이스로 변경한다.
- synonym 또는 권한 grant가 필요한 경우 WAS 계정 기준으로 추가한다.
- 이 DDL에는 운영 비밀번호나 LDAP 정보가 포함되지 않는다.
