# SMS V3 Claude Code 작업 규칙

이 문서는 Claude Code가 v3 BASE PROJECT 작업을 시작할 때 읽는 진입 문서다.

Codex와 동일한 기준을 적용한다. 상세 규칙은 `AGENTS.md`와 `docs/base/*.md`를 따른다.

## 필수 확인

- `docs/base/README.md`를 먼저 읽는다.
- 읽기 순서는 `docs/base/rules-index.md`를 단일 기준으로 따른다.
- 작업 종류에 맞는 `.claude/rules/*.md`를 함께 확인한다.
- 운영(prod) 서버 또는 운영 DB 작업은 사전 승인 없이 진행하지 않는다.
- 지시받지 않은 패턴/라이브러리는 임의로 도입하지 않는다.
- 사용자/부서는 `EMP`, `DEP` 기준으로 설계한다.
- `EMP.PERM_*`는 v3 권한 판단에서 사용하지 않는다.
- local/dev/prod의 차이는 인증 방식만 다르며, 권한 판단은 공통 테이블 기준이다.
- 메뉴 테이블은 DBA가 생성할 수 있도록 DDL 문서와 seed 문서를 함께 관리한다.
- 메뉴 source 정책은 `docs/base/menu-source-policy.md`를 따른다.

## 낮은 모델용 작업 방식

판단을 길게 추론하지 말고 문서에 적힌 순서대로 실행한다.

1. 요구사항을 문서의 정책과 대조한다.
2. 불명확한 항목은 임의 구현하지 않고 미확정으로 기록한다.
3. 생성 후 검증 명령을 실행한다.
4. 실패하면 원인을 수정하고 같은 검증을 반복한다.
5. 같은 오류가 3회 반복되면 원문 오류, 시도한 3가지, 추정 원인을 보고한다.
